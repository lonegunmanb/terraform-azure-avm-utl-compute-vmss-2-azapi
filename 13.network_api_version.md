## Critical Review & Edge Case Analysis

### 1. Null Semantics

**When variable is null:**
- `coalesce()` applies default value "2020-11-01"
- Field is always set in request body (never omitted)

**Why this matches provider behavior:**
- Provider has `Default: NetworkApiVersionTwoZeroTwoZeroNegativeOneOneNegativeZeroOne`
- Provider always sends this field with default value when not explicitly set

### 2. Legacy Mode Handling

**Legacy mode condition:** `sku_name` is not set

**Behavior in legacy mode:**
- Main body still sets `networkApiVersion` (networkProfile is created)
- `post_creation_updates` list is empty (conditional returns null)
- DiffSuppressFunc suppresses changes when:
  - Old value is empty
  - New value is default "2020-11-01"

**Rationale:** Legacy VMSS resources may not have virtualMachineProfile. The suppress logic prevents spurious updates when migrating from legacy to new format.

### 3. Update Behavior

**Scenario 1: sku_name is null (legacy mode)**
- No post_creation_updates entry
- Main body sets networkApiVersion
- ignore_changes prevents drift detection

**Scenario 2: sku_name is set, value changes**
- Trigger evaluates suppress condition
- If not suppressed AND values differ → trigger is non-null → update fires
- If suppressed OR values match → trigger is null → no update

**Scenario 3: First apply with sku_name set, default value**
- existing_network_api_version = ""
- new_network_api_version = "2020-11-01"
- suppress = false (because sku_name != null)
- Trigger is null (because values don't differ after initial creation)

### 4. Integration with network_interface

**Conditional rendering:**
```hcl

var.orchestrated_virtual_machine_scale_set_sku_name != null ? {
```

**Purpose:** 
- Creates `networkProfile` when EITHER `network_interface` OR `sku_name` is set
- Ensures `networkApiVersion` is included even when only `sku_name` is set (non-legacy mode)
- When both are null (legacy mode), entire networkProfile is omitted

### 5. Idempotency

**First apply:**
- Creates resource with networkApiVersion = "2020-11-01" (default)
- existing_network_api_version starts as ""

**Subsequent applies (no changes):**
- existing_network_api_version = "2020-11-01"
- new_network_api_version = "2020-11-01"
- Trigger is null (values match)
- No update triggered ✅

**Value change:**
- User changes variable to "2022-11-01"
- new_network_api_version = "2022-11-01"
- suppress = false (values differ and not suppressed)
- Trigger is non-null → update fires ✅

### 6. Boundary Conditions

**Empty string in state:**
- Handled by default "" in `try()` expression
- Correctly identified as "not set"

**Invalid API version:**
- Blocked by validation in variables.tf before reaching locals
- Error message guides user to correct values

**sku_name changes from null to value:**
- First apply: legacy mode, no post_creation_updates
- Subsequent apply: non-legacy mode, post_creation_updates created
- May trigger update if version differs from state
