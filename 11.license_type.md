## Special Handling

### Validation

**Implemented in variables.tf:**

```hcl
validation {
  condition = (
    var.orchestrated_virtual_machine_scale_set_license_type == null ||
    contains(["None", "Windows_Client", "Windows_Server"], var.orchestrated_virtual_machine_scale_set_license_type)
  )
  error_message = "The license_type must be one of 'None', 'Windows_Client', or 'Windows_Server'."
}
```

**Rationale:** Replicates the provider's `validation.StringInSlice` validation to fail fast at plan time rather than during API call.

### DiffSuppressFunc Replication

The provider has a `DiffSuppressFunc` that treats "None" and empty string as equivalent:

```go
DiffSuppressFunc: func(_, old, new string, _ *pluginsdk.ResourceData) bool {
    if old == "None" && new == "" || old == "" && new == "None" {
        return true
    }

    return false
},
```

**Implementation Strategy:**

Instead of directly replicating the diff suppression logic (which would be complex in HCL), we achieve the same behavior by:
1. Only including `licenseType` in the body when the value is not null AND not "None"
2. This means both `null` and `"None"` result in the field being omitted from the body
3. Azure API will treat the omitted field as "None" (the default)

**Implementation:**

```hcl
var.orchestrated_virtual_machine_scale_set_license_type != null && var.orchestrated_virtual_machine_scale_set_license_type != "None" ? {
  licenseType = var.orchestrated_virtual_machine_scale_set_license_type
} : {}
```

**Behavior Analysis:**
- When `license_type = null`: Field omitted from body (equivalent to "None")
- When `license_type = "None"`: Field omitted from body (matches provider behavior)
- When `license_type = "Windows_Client"`: Field included with value "Windows_Client"
- When `license_type = "Windows_Server"`: Field included with value "Windows_Server"

This exactly replicates the provider's behavior where "None" and "" (omitted) are treated as equivalent.

- **Empty string:** Not a valid input (validation prevents it)
