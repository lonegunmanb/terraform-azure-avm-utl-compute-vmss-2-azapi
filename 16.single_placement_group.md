### 2. Conditional ForceNew - Update Phase Restriction

From `resourceOrchestratedVirtualMachineScaleSetUpdate`:

```go
if d.HasChange("single_placement_group") {
    // Since null is now a valid value for single_placement_group
    // make sure it is in the config file before you set the value
    // on the update props...
    if !pluginsdk.IsExplicitlyNullInConfig(d, "single_placement_group") {
        singlePlacementGroup := d.Get("single_placement_group").(bool)
        if singlePlacementGroup {
            return fmt.Errorf("`single_placement_group` cannot be changed from `false` to `true`")
        }
        updateProps.SinglePlacementGroup = pointer.To(singlePlacementGroup)
    }
}
```

**Key Behavior:**
- ✅ `true` → `false`: Allowed (Update in place)
- ✅ `true` → `null`: Allowed (Update in place)
- ✅ `false` → `null`: Allowed (Update in place) 
- ✅ `null` → `false`: Allowed (Update in place)
- ✅ `null` → `true`: Allowed (Update in place)
- ❌ `false` → `true`: FORBIDDEN - Must trigger replacement

**Reasoning:** Once a VMSS is configured with `single_placement_group = false`, enabling single placement group (`true`) would require recreating the resource because it affects the underlying placement architecture.

**Shadow Implementation:**


**Logic Explanation:**

- The `value` field contains the boolean condition result, not the data itself

### Edge Cases

1. **`null` → `true` transition:**
   - Existing: `null` (not in state)
   - New: `true`
   - `try()` returns `null`, condition `false && true` = `false`
   - ✅ Correctly allowed (no replacement)

2. **`null` → `false` transition:**
   - Existing: `null`
   - New: `false`
   - Condition `false && false` = `false`
   - ✅ Correctly allowed (no replacement)

3. **`true` → `false` transition:**
   - Existing: `true`
   - New: `false`
   - Condition `false && false` = `false`
   - ✅ Correctly allowed (no replacement)

4. **`false` → `true` transition (FORBIDDEN):**
   - Existing: `false`
   - New: `true`
   - Condition `false && true` = `true`
   - ✅ Correctly triggers replacement

5. **`false` → `null` transition:**
   - Existing: `false`
   - New: `null`
   - Condition `false && null` (null treated as non-true) = `false`
   - ✅ Correctly allowed (no replacement)

6. **`true` → `null` transition:**
   - Existing: `true`
   - New: `null`
   - Condition `false && null` = `false`
   - ✅ Correctly allowed (no replacement)
